# 架构设计文档

## 系统架构

```
┌─────────────────────────────────────────────────────────────┐
│                        前端 (Frontend)                        │
│                     React + TypeScript                       │
│  ┌─────────────────────────────────────────────────────┐   │
│  │              KnowledgeSelector 组件                  │   │
│  │  - 知识点选择 (带前置依赖自动处理)                    │   │
│  │  - 难度区间设置                                     │   │
│  │  - 题型选择                                         │   │
│  └─────────────────────────────────────────────────────┘   │
└──────────────────────────┬──────────────────────────────────┘
                           │ HTTP
┌──────────────────────────▼──────────────────────────────────┐
│                      Nginx 反向代理                           │
│  - 静态文件服务                                              │
│  - 路由转发                                                  │
└──────────────────────────┬──────────────────────────────────┘
                           │
┌──────────────────────────▼──────────────────────────────────┐
│                        后端 (Backend)                         │
│                         Python                               │
│  ┌─────────────────────────────────────────────────────┐   │
│  │                   MCTS 算法核心                      │   │
│  │  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐ │   │

└─────────────────────────────────────────────────────────────┘
```

## 模块说明

### Frontend
- **KnowledgeSelector**: 知识点选择器 React 组件
- **App**: 主应用组件，处理路由和状态

### Backend
- **main.py**: 程序入口，协调整个流程
- **knowledge_selector.py**: 知识点选择 GUI (tkinter)
- **question_node.py**: MCTS 节点和搜索算法实现
- **llm_client.py**: DeepSeek API 客户端
- **prompt_templates_*.py**: LLM 提示词模板

## 数据流

1. 用户在前端选择知识点、难度、题型
2. 前端将选择结果发送到后端 (或直接运行 Python 程序)
3. 后端初始化 MCTS 根节点
4. MCTS 循环执行 Select → Expand → Simulate → Backpropagate
5. 生成高质量题目并保存到文件
