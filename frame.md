# 高数题目生成器框架设计

## 1. 核心思路

通过蒙特卡洛树搜索（MCTS）与大型语言模型（LLM）的结合，自动生成高质量的综合性高等数学题目。

**关键洞察**：
- 按知识点拓扑顺序逐层扩展，每层加入一个知识点
- 树结构提供随机性和创新性（同一父节点可生成多个子节点）
- 通过UCT评分选择高质量路径，剪枝低质量分支
- 保留高质量中间节点作为可用题目

## 2. MCTS适配思路

### 2.1 概念映射

| MCTS概念 | 题目生成映射 |
|---------|------------|
| 节点 | 包含特定知识点集合的题目 |
| 根节点 | 空节点（无知识点） |
| 子节点 | 加入新知识点后的聚合题目 |
| 叶节点 | 知识点队列为空的终止题目 |
| 扩展 | 从题库选取知识点题目，与当前节点聚合 |
| 选择 | UCT评分选择高质量节点扩展 |
| 剪枝 | 只保留top-k高质量节点 |

### 2.2 树扩展方式

按拓扑排序的知识点队列逐层扩展：

```
                    [空]
                   / | \
                 A1 A2 A3   ← 第一层：加入知识点A的多种可能
                /|\ /|\ /|\
              B1 B2 B3...   ← 第二层：加入知识点B的多种可能
              /|\ 
            C1 C2 C3...     ← 第三层：继续扩展
```

每层通过多种方式生成子节点（不同题库题目、不同聚合策略），但只保留高质量分支。

## 3. 系统架构

```
用户交互层
    ├── 知识点选择（多选）
    ├── 难度目标设定
    └── 参考题集上传（可选）
            ↓
题目生成引擎
    ├── 拓扑排序模块（知识点排序）
    ├── MCTS搜索核心
    │   ├── 选择：UCT策略
    │   ├── 扩展：聚合生成
    │   ├── 评估：质量评分
    │   └── 回溯：更新路径
    └── 剪枝控制（top-k保留）
            ↓
LLM服务层
    ├── 题目聚合（多种策略）
    ├── 质量评估（分层评估）
    └── 解答生成（可选）
            ↓
数据层
    ├── 知识点图谱（拓扑关系）
    ├── 参考题库
    └── 生成结果存储
```

## 4. 关键设计问题

### 4.1 子节点生成方式

每个节点扩展时，需要考虑：
- 从题库中选取该知识点的哪些题目进行聚合
- 采用何种聚合策略（串联、并联、嵌套等）
- 是否引入LLM随机性生成变体

### 4.2 评估策略分层

建议采用三层评估降低成本：

1. **快速筛选**：结构特征（长度、公式数、知识点覆盖率）
2. **质量评估**：与参考题相似度、轻量级LLM评分
3. **精细验证**：完整LLM评估、解答生成验证（仅终止节点）

### 4.3 剪枝与多样性平衡

- 每层保留top-k节点防止爆炸
- 但需保证多样性（不同聚合路径）
- 可考虑按质量分档，每档保留一定数量的节点

## 5. 知识点体系

### 5.1 标签分类（示例）

```
高等数学
├── 极限与连续
├── 微分学
├── 积分学
├── 级数
├── 微分方程
└── 多元函数
```

### 5.2 拓扑关系

需要维护知识点间的前置依赖：
- 不定积分 → 定积分 → 重积分
- 导数定义 → 微分中值定理
- 等等

## 6. 待探索问题

1. **聚合策略的具体设计**：串联、并联、嵌套的具体实现方式
2. **评估指标的有效性**：哪些指标真正反映题目质量
3. **LLM调用成本控制**：如何减少token消耗同时保证质量
4. **终止条件的选择**：何时停止扩展，如何选出最终题目
5. **参考题集的利用方式**：RAG、相似度匹配、风格迁移等

## 7. 实现阶段

### Phase 1: 验证核心假设
- 实现基础树结构
- 验证LLM聚合可行性
- 建立简单评估体系

### Phase 2: 完善搜索策略
- 优化UCT评分机制
- 实现分层评估
- 添加剪枝控制

### Phase 3: 质量优化
- 引入参考题集学习

- 添加解答生成

### Phase 4: 工程化
- 并行搜索优化
- 可视化生成过程
